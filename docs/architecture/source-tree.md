# Source Tree

## Turborepo Structure

The project will be organized as a Turborepo monorepo.

```
/cleanly-app
├── apps
│   ├── web/                # The Next.js frontend application
│   └── supabase/           # The Supabase backend project
│       ├── functions/      # Supabase Edge Functions
│       ├── migrations/     # SQL migration files generated by Prisma
│       └── config.toml     # Core Supabase project configuration
│
├── packages
│   ├── db/                 # Prisma schema and generated client
│   ├── types/              # Shared TypeScript interfaces
│   └── ui/                 # Shared UI components
│
├── package.json
└── turbo.json
```

## Developer Workflow

The database schema is defined in `packages/db/prisma/schema.prisma`. Developers will use Prisma to generate SQL migration files, which are placed in `apps/supabase/migrations/` to be deployed by the Supabase CLI.
